Function: Функция: 
  Initialize AWS tape image. Инициализация ленты AWS.
 
Syntax:Формат: 
 
  AWSINIT [options] awstape volser [owner]
 
Options:Режимы: 
 -SL | -NL
 
PrerequestedТребования: 
 
  • Either Open object REXX (OOREXX) or Regina REXXOpen Object REXX (OOREXX) либо Regina REXX
  • File extension .rex must be properly associatedРасширение имени файлов .rex должно быть правильно ассоциировано
 
InstallationУстановка 
  Installation procedure not required. Какая-либо специальная установка не требуется.
 
Usage:Применение 
 
  1. Procedure AWSINIT do principally the same as z/OS IEHINITT does. By default, or if -SL option specified, 2 parameters required: name of AWS tape image awstape, volume serial volser. 3rd parameter owner is optional.
    If option -NL specified, single parameter awsfile required.     Функции процедура AWSINIT в принципе соответствует функции программы IEHINITT z/OS. По умолчанию и при явном указании параметра -SL требуется два параметра: имя образа ленты AWS awstape и серийный номер тома volser. В этом случае может указываться необязательный 3ий пармаметр owner.
    Если указывается режим -NL, требуется только один параметр awsfile.
  2. When option -SL in effect, AWSINIT write to AWS tape standard volume label VOL1, dummy label HDR1 and tape mark, the same as IEHINITT does. Если действует режим -SL, AWSINIT записывает на ленту AWS стандартную метку тома VOL1, фиктивную метку HDR1 и ленточную марку, т.е. точно то же самое, что делает IEHINITT.
  3. When option -NL in effect, AWSINIT write to AWS tape single tape mark. Если действует режим -NL, AWSINIT записывает на ленту AWS одну ленточную марку.
  4. Warning: any content of existing AWS tape would be completely destroyed by AWSINIT. Предупреждение: предыдущее содержимое ленты AWS после применения AWSINIT полностью уничтожается.
