Function: Функция: 
  List and extract member(s) from XMIT (TRANSMIT, IDTF) image. Список разделов и их извлечение из файла, выгруженного XMIT (TRANSMIT, IDTF).
 
Syntax:Формат: 
 
  UNXMIT [options] xmit [{member | *} [rename_pattern]]
 
Options:Режимы: 
 -binary | -text
-ebcdic codepage
-ascii codepage
 
PrerequestedТребования: 
 
  • Either Open object REXX (OOREXX) or Regina REXXOpen Object REXX (OOREXX) либо Regina REXX
  • File extension .rex must be properly associatedРасширение имени файлов .rex должно быть правильно ассоциировано
 
InstallationУстановка 
  Installation procedure not required. Какая-либо специальная установка не требуется.
 
Usage:Применение 
 
  1. Procedure UNXMIT has single mandatory parameter xmit - name of XMIT image. An optional 2nd parameter could specify member to be processed or * to process all members, 3rd parameter specify rename pattern.
    When 2nd and 3rd parameters are missed procedure lists all member names from XMIT image file:     Процедура UNXMIT имеет один обязательный параметр ,i>xmit - имя файла с образом набора данных, выгруженного XMIT. Второй параметр может указывать имя обрабатываемого раздела или *, что указывает обработку всех разделов.
    Если второй параметр отсутствует, процедура отображает список разделов выгруженного библиотечного набора данных:
    F:\>unxmit.rex FILE845.XMI
    021I Data set 'CBT.V495.FILE845.PDS' from SBGOLOB at N1
    020I Member $$$#DATE TTR 000204
    020I Member $README TTR 000206
    020I Member @FILE845 TTR 000208
    020I Member HOSTCMD TTR 00020A
    020I Member HOSTCMD$ TTR 00020E
    020I Member HOSTCMD# TTR 000210
    020I Member IKJTSO00 TTR 000212
    020I Member RUNAUTH TTR 000304
    020I Member RUNAUTH$ TTR 000308
    020I Member RUNAUTH# TTR 00030A
    020I Member RUNAUTH@ TTR 00030C
    020I Member SYSCMD TTR 00030E
    020I Member SYSCMD$ TTR 000311
    020I Member SYSCMD# TTR 000313

  2. File name of the extracted member(s) build using rename pattern, where %D% substituted with original data set name and %M% substituted with original member name. If rename pattern is not specified, default pattern %D%(%M%) is used, and file would be original_data_set_name(original_member_name) Имя файла с извлекаемым разделом (разделами) формируется на основании шаблона переименования, при этом %D% заменяется именем оригинального набора данных, а %M% заменяется именем оригинального раздела. Если шаблон переименования не был указан, по умолчанию используетяс шаблон %D%(%M%), и имя файла будет оригинальное_имя_набора_данных(оригинальное_имя_раздела)
    So, in example shown above, file names would be: Так, в примере приведенном выше, имена файлов будут: CBT.V495.FILE845.PDS($$$#DATE)
    CBT.V495.FILE845.PDS($README) and so on. и т.д.
    Command Команда unxmit.rex FILE845.XMI * %m%.txt extracts members with names извлечет разделы с именами $$$#DATE.txt
    $README.txt
    etc.
    For the sequential data set default pattern is %D%, so file name would be an original data set name. Для последовательного набора данных шаблон по умолчанию %D%, так что именем файла будет имя оригинального набора данных.
  3. Data translation. If option -text specified (this is the default option), data would be translated from EBCDIC to ASCII. The options -ascii and -ebcdic may be used to specify a particular code pages for translation. If both -ascii and -ebcdic are missed, an internal translation table is used. This internal table specified 1025 to 1251 translation.
    If option -binary specified, original data remains unchanged.     Перекодировка данных. Если указан режим -text (этот режим активен по уолчанию), option), данные транслируются из кода EBCDIC в код ASCII. Режимы -ascii и -ebcdic могут использоваться чтобы определить, какие именно кодовые таблицы используются при перекодировке. Если режимы -ascii и -ebcdic не указаны, используется внутренная таблица перекодировки, которая определяет перекодировку из 1025 в 1251.
    Если указан режим -binary оригинальные данные остаются неизменными.
    F:\>unxmit.rex FILE845.XMI $readme
    021I Data set CBT.V495.FILE845.PDS from SBGOLOB at N1
    018I Member $README extracted to CBT.V495.FILE845.PDS($README)

  4. XMIT image is very popular format for z/OS user-written software distribution. For instance, CBTTAPE (https://cbttape.org) files distributed in XMIT format. Формат XMIT - очень популярный формат, используемый для распространия программного обеспечения z/OS, написанного пользователями. В частности, все файлы, распространяемые CBTTAPE (https://cbttape.org) имеют формат XMIT.
