Function: Функция: 
  Browse XMIT archive - browse unloaded sequuential data set as well as members of unloaded PDS[E]. xmitlist implemented as Windows console application and designed in old fashion VM/CMS FILELIST style :-) Просмотр содержимого файла XMIT. Отображается содержимое выгруженного последовательного набора данных либо содержимое разделов выгруженного библиотечного набора данных (PDS[E]). xmitlist реализован как консольное приложение Windows, а его внешний вид и функциональность подобны старомодному VM/CMS FILELIST :-)
 
Syntax:Формат: 
 
  XMITLIST [-ebcdic codepage] [-ascii codepage]xmitfile
 
PrerequestedТребования: 
 
  • Either Open object REXX (OOREXX) or Regina REXXOpen Object REXX (OOREXX) либо Regina REXX
  • File extension .rex must be properly associatedРасширение имени файлов .rex должно быть правильно ассоциировано
  • The Hessling Editor (THE), Windows version 3.3B3 includedThe Hessling Editor (THE), версия Windows 3.3B3 включена
 
InstallationУстановка 
  Special installation procedure not required. However, windows console session used for xmitlist need to be customized. xmitlist provide browsing of text data, so it is important to represent it correctly, including possible national characters. xmitlist delivered with shortcut (xmitlist.lnk) already customized for cyrillic languages. If you use different language, you need to change xmitlist propeties as follows:
  • specify the proper font which is represent national characters
  • specify correct EBCDIC and ASCII codepages (-e and -a options) to provide correct national characters representation. Please see this page and choose a proper codepage for your native language. Please use THE against any file with national characters to be sure that ASCII console codepage is correct.

Delivered shortcut defines font "Lucida Console" with 14px size, and do not specified -e and -a options, therefore default internal translate table from codepage 1025 (EBCDIC cyrillic, it is practically the same as 20880) and 866 (Windows OEM cyrillic) are used.

Anyway, path to program object and working directory specified in shortcut must be changed to reflect actual location

 Какая-либо специальная установка xmitlist не требуется, однако, может потребоваться настройка сессии консоли windows. Так как основной функцией xmitlist является просмотр данных, они должны корректно отображаться, включая национальные символы. xmitlist поставляется с ярлыком (xmitlist.lnk), который обеспечивает отображение символов кириллицы. Если для представления данных используется другой язык, необходимо изменить свойства ярлыка следующим образом:
  • указать подходящий шрифт для правильного представления национальных символов;
  • указать подходящие кодовые страницы EBCDIC и ASCII для правильного представления национальных символов. См. эту страницу для выбора подходящих кодовых страниц используемого языка. Используйте THE для редактирования произвольного файла с национальными символами, чтобы убедиться в правильности кодовой страницы ASCII.

Поставляемый ярлык определяет шрифт "Lucida Console" размером 14px и не определяет режимы -e и -a, поэтому используется встроенная таблица трансляции из 1025 ((EBCDIC кириллица, практически совпадает с 20880) и 866 (Windows OEM кириллица).

В любом случае, путь к программному объекту и рабочая папка в ярлыке должны быть изменены, чтобы отразить фактическое местонахождение xmitlist.

On the top shown script name, it's current version, an original data set name and name of processed XMIT file (FILE020.XMI here).
For a partitioned data set (PDS[E]) member list is shown, for a sequential data set content of the data set displayed.
You can scroll the member list with F7/F8 and PgUp/PgDown, but also can unhide command line with F2 and then use any THE command - locate, find etc.
Move cursor to the first member in list (member $$$DOC) and press F10. Content of the member $$$DOC being displayed: В верхней строке отображается имя программы и ее текущая версия, имя оригинального набора данных и имя обрабатываемого файла.
Список можно листать с помощью F7/F8 и PgUp/PgDown, но также можно вызвать командную строку THE с помощью F2 и затем использовать любые команды THE - locate, find etc.

/****************************** REXX *******************************/
/* THIS REXX PROGRAM LISTS ALL THE APF LIBRARIES BY BROWSING */
/* THE AUTHORIZED LIBRARY TABLE AT OFFSET 484 IN THE CVT. */
/* THE LAYOUT IS: NUMBER OF ENTRIES 2 */
/* LEN OF ENTRY 1 ) TIMES */
/* VOLUME 6 ) NUMBER */
/* DSN VARIABLE ) OF ENTRIES */
/*********************************************************************/
ARG NAME
CLEAR /* CLEAR SCREEN */
CVT = STORAGE(10,4) /* DECIMAL 16 ==> CVT */
CCVT = C2X(CVT) /* CHAR CVT ADDRESS */
DCVT = C2D(CVT) /* DECIMAL CVT ADDR */
DAUTH = DCVT + 484 /* DECIMAL AUTH ADDR */
AUTH = D2C(DAUTH,4) /* AUTH TAB ADDRESS */
CAUTH = C2X(AUTH) /* CHAR AUTH TAB ADDR */
ADDR1 = STORAGE(CAUTH,4) /* ADDRESS OF AUTH TAB */
CADDR1 = C2X(ADDR1) /* CHAR ADDR OF AUTH */
DATA = STORAGE(CADDR1,2) /* GET LUMP OF DATA */
NUMENT = SUBSTR(DATA,1,2) /* NUMBER OF ENTRIES */
DNUMENT = C2D(NUMENT) /* CONVERT TO DECIMAL */
LEN = (DNUMENT * 51) + 2 /* MAX STORAGE POSS */

As well as F10, there are available Alt+F10 - extract current member to the file in text mode (i.e. converted to ASCII) and Ctrl+F10 - extract current member to the file in binary mode (i.e. in EBCDIC). Наряду с F10 в списке разделов поддерживаются Alt+F10 - извлечение текущего раздела в файл в текстовом режиме (т.е. перекодированного в ASCII) и Ctrl+F10 - извлечение текущего раздела в файл в двоичном режиме (т.е. в EBCDIC, без перекодировки)
 
NotesПримечания  
 
  • Procedure kept on the current directory all browsed files and directories as well as member list. Процедура не удаляет файлы, появившиеся в результате просмотра.
  • Procedure may be considered as XMITMGR alternative.
    Advantages: XMITLIST works on any system, where OOREXX and THE 3.3 available;
    XMITLIST is true open source, you can adopt it as you want.
    Disadvantages: you need OOREXX and THE unlike XMITMSG which is standalone (not 100% true - runtime required anyway).     Процедура может рассматриваться как альтернатива XMITMGR.
    Преимущества: XMITLIST может функционировать в любой версии Windows, и вообще на любой платформе, где доступен OOREXX и THE 3.3 и выше;
    В отличие от XMITMGR, у XMITLIST открыт исходный код, и скрипт можно модифицировать в соотвестствии с Вашими потребностями, я лишь прошу сохранить copyright.
    Недостатки: XMITLIST требует OOREXX и THE, в то время как XMITMGR полностью автономен (ну не совсем, там runtime тоже требуется на самом деле).
