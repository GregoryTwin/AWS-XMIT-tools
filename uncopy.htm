Function: Функция: 
  List and load member(s) from IEBCOPY image. Список разделов и их загрузка из файла, выгруженного IEBCOPY.
 
Syntax:Формат: 
 
  UNCOPY [options] iebcopy [{member|*} [rename]]
 
Options:Режимы: 
 -binary | -text
-ebcdic codepage
-ascii codepage
 
PrerequestedТребования: 
 
  • Either Open object REXX (OOREXX) or Regina REXXOpen Object REXX (OOREXX) либо Regina REXX
  • File extension .rex must be properly associatedРасширение имени файлов .rex должно быть правильно ассоциировано
 
InstallationУстановка 
  Installation procedure not required. Какая-либо специальная установка не требуется.
 
Usage:Применение 
 
  1. Procedure UNCOPY has single mandatory parameter iebcopy - name of IEBCOPY image. An optional 2nd parameter could specify member to be processed or * to process all members. 3rd parameter if specified is a rename pattern.
    When 2nd and 3rd parameters are missed procedure lists all member names from IEBCOPY image file:.     Процедура UNCOPY имеет один обязательный параметр iebcopy - имя файла с образом набора данных, выгруженного IEBCOPY. Второй параметр может указывать имя обрабатываемого раздела или *, что указывает обработку всех разделов. Третий параметр, если он указан, это шаблон переименования разделов.
    Если второй и третий параметры отсутствует, процедура отображает список разделов выгруженного библиотечного набора данных:
    F:\>uncopy.rex MVSFILE
    022I Data set attributes: DSORG=PO,RECFM=VB,LRECL=255,BLKSIZE=8209
    020I Member $INDEXPD TTR 001104
    020I Member QM001181 TTR 000107

  2. File name of the extracted member(s) build using rename pattern, where %M% substituted with original member name. If rename pattern is not specified, default pattern %M% is used, and file would be an original member name. Имя файла с излекаемым разделом (разделами) формируется на основании шаблона переименования, при этом %M% заменяется именем оригинального раздела. Если шаблон переименования не был указан, по умолчанию используется шаблон %M%, и именем файла будет оригинальное имя раздела.
    So, command Так, команда uncopy.rex MVSFILE * extract members with file names $INDEXPD and QM001181 извлечет разделы с именами $INDEXPD и QM001181
    Command Команда uncopy.rex MVSFILE * %m.txt extract members with file names $INDEXPD.txt and QM001181.txt извлечет разделы с именами $INDEXPD.txt и QM001181.txt
  3. Data translation. If option -text specified (this is the default option), data would be translated from EBCDIC to ASCII. The options -ascii and -ebcdic may be used to specify a particular code pages for translation. If both -ascii and -ebcdic are missed, an internal translation table is used. This internal table specified 1025 to 1251 translation.
    If option -binary specified, original data remains unchanged.     Перекодировка данных. Если указан режим -text (этот режим активен по умолчанию), данные транслируются из кода EBCDIC в код ASCII. Режимы -ascii и -ebcdic могут использоваться чтобы определить, какие именно кодовые таблицы используются при перекодировке. Если режимы -ascii и -ebcdic не указаны, используется внутренная таблица перекодировки, которая определяет перекодировку из 1025 в 1251.
    Если указан режим -binary оригинальные данные остаются неизменными.
    F:\>uncopy.rex -b -s pdf MVSFILE QM001181
    022I Data set attributes: DSORG=PO,RECFM=VB,LRECL=255,BLKSIZE=8209
    018I Member QM001181.pdf extracted
    F:\>uncopy.rex -s txt MVSFILE $INDEXPD
    022I Data set attributes: DSORG=PO,RECFM=VB,LRECL=255,BLKSIZE=8209
    018I Member $INDEXPD.txt extracted

  4. The most useful UNCOPY usage is a processing of IBM ESD (Electronic Software Delivery) files. Such files produced by SMP/E GIMZIP utility and look like S0005.CSP.STP99999.PGMDIR.pax.Z. To process data on the PC you need first extract file called MVSFILE from archieve using 7z or it equivalent (uncompress and tar). Then MVSFILE could be processed with UNCOPY, because MVSFILE is a IEBCOPY image. Наиболее частое применение UNCOPY это обработка файлов поставки IBM ESD (Electronic Software Delivery). Такие файлы построены утилитой SMP/E GIMZIP и выглядят как S0005.CSP.STP99999.PGMDIR.pax.Z. Чтобы обработать данные необходимо вначале извлечь файл с именем MVSFILE из архива. Это можно сделать с помощью программы 7z или ее эквивалента (uncompress и tar). После этого файл MVSFILE можно обработать UNCOPY, так как MVSFILE представляет собой образ набора данных, выгруженного IEBCOPY.
